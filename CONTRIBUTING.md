# Contributing to glance-cli

Thanks for your interest in contributing! We welcome contributions of all kinds.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
```bash
   git clone https://github.com/jkenley/glance-cli.git
   cd glance-cli
```
3. **Install dependencies**:
```bash
   bun install
```
4. **Create a branch** for your changes:
```bash
   git checkout -b feature/your-feature-name
```

## Development Workflow

- **Run locally**: `bun dev https://example.com`
- **Run tests**: `bun test`
- **Build**: `bun build`

## Before Submitting

- [ ] Code follows existing style and conventions
- [ ] Tests added/updated if applicable
- [ ] Documentation updated if needed
- [ ] All tests pass
- [ ] Commits are clear and descriptive

## Submitting Changes

1. **Push to your fork**:
```bash
   git push origin feature/your-feature-name
```
2. **Open a Pull Request** on GitHub
3. **Describe your changes** clearly in the PR description

## Reporting Issues

For bugs or feature requests:
- Check if it already exists in [Issues](https://github.com/jkenley/glance-cli/issues)
- If not, open a new issue with:
  - Clear description
  - Steps to reproduce (for bugs)
  - Expected vs actual behavior
  - Your environment (OS, Node/Bun version, etc.)

## Questions?

Feel free to open an issue for discussion or reach out to **alo@jkenley.me**.
