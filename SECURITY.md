# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in **glance-cli**, please report it responsibly.

### How to Report

Send an email to: **alo@jkenley.me**  
Subject: `[Security] <brief description>`

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

We will acknowledge receipt within 48 hours and work with you on disclosure timing.

### Supported Versions

Only the latest released version receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

### Disclosure Policy

- We will investigate all legitimate reports
- If confirmed, we will release a patch as quickly as possible
- Credit will be given to reporters (unless anonymity requested)
- Security advisories will be published on GitHub

### Security Best Practices

When using glance-cli:
- Keep your installation up to date
- Protect your API keys (use environment variables, not hardcoded values)
- Be cautious when processing untrusted URLs
- Review exported content before sharing
